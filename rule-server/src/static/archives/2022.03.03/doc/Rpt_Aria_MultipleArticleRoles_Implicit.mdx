### Multiple elements with `"article"` role do not have unique labels

<div id="locLevel"></div>

Each element with an `"article"` role must have a unique label that describes its purpose

</Column>

<Row>
<Column colLg={11} colMd={5} colSm={4} className="toolMain">

### Why is this important?

An HTML `<article>` element, or any element with an explicit `"article"` role is a type of ARIA landmark region. When there are multiple article regions, they must have unique labels, so people who use assistive technologies can quickly identify and navigate to each `"article"` region.

<div id="locSnippet"></div>

### What to do

 * For each `<article>` element or element with `role="article"`, either set the value of the `aria-labelledby` attribute of each article to the `id` value of any visible text that appropriately and uniquely describes that article's purpose;
 * **Or**, use the `aria-label` attribute to add an invisible, unique, non-empty label to each element with the `"article"` role.

For example:

<CodeSnippet type="multi" light={true}>&lt;article aria-label="weather portlet"&gt;
    ...
&lt;/article&gt;
    
&lt;div role="article" aria-labelledby="t1"&gt;
    &lt;h1 id="t1"&gt;Stock Ticker Portlet&lt;/h1"&gt;
    ...
&lt;/div&gt;</CodeSnippet>

</Column>

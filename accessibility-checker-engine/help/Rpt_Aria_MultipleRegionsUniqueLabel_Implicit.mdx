### Multiple elements with `"region"` role do not have unique labels

Multiple elements with `"region"` role must have unique labels

<Column colLg={11} colMd={5} colSm={4} className="toolMain">

### Why is this important?

When there are multiple elements with a `"region"` role, they must have unique labels that clearly describe the region's content. This includes labelled `<section>` elements. This label helps people using assistive technologies to quickly find and navigate to each region.

<div id="locSnippet"></div>

### What to do

* For each element with a `"region"` role, either set the value of the `aria-labelledby` attribute of each region to the `id` value of any visible text that appropriately and uniquely describes that region's content;
* OR, use the `aria-label` attribute to add an invisible, unique, non-empty label to each element with the `"region"` role.

For example:

<CodeSnippet type="multi" light={true}>&lt;div role="region" aria-label="Weather portlet"&gt;
...
&lt;/div&gt;
    
&lt;div role="region" aria-label="Stock ticker portlet"&gt;
...
&lt;/div&gt;</CodeSnippet>

</Column>

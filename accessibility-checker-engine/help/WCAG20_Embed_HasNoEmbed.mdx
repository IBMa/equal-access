### Why is this important?

The `<noembed>` element is used to provide information for screen readers to display when the browser cannot display an embedded object. The content of the `<noembed>` element is rendered only if the content of the `<embed>` element is not supported. It must be positioned so as to be clear to assistive technologies that a text alternative is associated with the `<embed>` element it describes.

<div id="locSnippet"></div>

### What to do

 * The `<noembed>` element is deprecated and should be avoided. Use the `<object>` element instead of `<embed>` and use the body of the element to provide fallback content when the object does not load.

  For example:

<CodeSnippet type="multi" light={true}> &lt;object src="../movies/history_of_rome.mov" height="60" width="144" autostart="false"&gt;
         &lt;a href="../transcripts/transcript_history_rome.htm"&gt;Transcript of "The history of Rome"&lt;/a&gt;
 &lt;/object&gt;
 </CodeSnippet>

### About this requirement

[IBM 1.1.1 Non-text content](https://www.ibm.com/able/requirements/requirements/#1_1_1)
[WCAG 2.1 technique H53](https://www.w3.org/WAI/WCAG21/Techniques/html/H53)
[WCAG 2.1 technique H46](https://www.w3.org/WAI/WCAG21/Techniques/html/H46.html)

### Who does this affect?

 * Blind people using a screen reader
 * People using text only, monochrome or braille displays
 * People using text-based browsers (e.g., Lynx) or audio interfaces
 * People who turn off image-loading on their web browsers

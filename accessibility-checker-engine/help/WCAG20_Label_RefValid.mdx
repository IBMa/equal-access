## Who does this affect?

* Blind people using screen readers
* People with low vision using screen readers
* People with dexterity impairment using voice control

## Why is this important?

A `label` element is attached to a specific `<input>` element through the use of the `"for"` attribute. When a `<label>` does not reference the corresponding unique `"id"` in a valid lableable `<input>` element, assistive technologies will not be able to associate the label with the element.

</Column>

<div id="locLevel" className="level-violation">Violation</div>

## The `<label>` `"for"` attribute value of \"{0}\" is not the `"id"` of a valid `<input>` element
The `"for"` attribute must reference a non-empty, unique `"id"` attribute of an `<input>` element

[IBM 1.3.1 Info and Relationships](https://www.ibm.com/able/guidelines/ci162/info_and_relationships.html) | [WCAG 2.1 technique H44](https://www.w3.org/WAI/WCAG21/Techniques/html/H44)

<div id="locSnippet"></div>

## What to do

 * Set the value of the `"for"` attribute to the `"id"` of the valid `<input>` element to be labeled. Make sure the `"id"` is unique. For example:

```
<label id="label2" for="ln1">Last Name</label>
<input type="text" name="lastname" id="ln1" value="enter last name">
```

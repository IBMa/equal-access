## Who does this affect?

* Blind people using screen readers

## Why is this important?

A element with `"contentinfo"` role is an area that identifies common information at the bottom of each page, typically called the footer of the page, including information such as copyrights and links to privacy and accessibility statements. It is logical that a footer without the main content within a `"main"`role is not permited.
HTML `<footer>` element in context of body as an implicit role of `"contentinfo"`.
</Column>

## A element with `"contentinfo"` role is present without an element with a `"main"` role
A element with `"contentinfo"` role is only permitted with an element with `"main"` role

[IBM 2.4.1 Bypass Blocks](http://www.ibm.com/able/guidelines/ci162/bypass_blocks.html) | [WAI-ARIA 1.1 Contentinfo Role](https://www.w3.org/TR/wai-aria-1.1/#contentinfo)

<div id="locSnippet"></div>

## What to do

* Add a `<main>` element that has an implicit role of `"main"` to enclose the main content of the Web page.


For example:
```
<body>
    <div role="main">
      ...
    </div>
    <div role="contentinfo">
	    ...
    </div>
</body>
```

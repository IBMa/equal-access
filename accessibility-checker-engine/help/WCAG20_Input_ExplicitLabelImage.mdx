## Who does this affect?

* Blind people using screen readers
* Peopl with low vision using screen readers
* People using text-based browsers (e.g., Lynx) or audio interfaces

## The `<input>` element of type `"image"` has no text alternative
 The `<input>` element of type `"image"` should have a text alternative

[IBM 1.1.1 Non-text Content](http://www.ibm.com/able/guidelines/ci162/non_text_content.html) | [WCAG 2.1 HTML technique H36](https://www.w3.org/WAI/WCAG21/Techniques/html/H36)

<div id="locSnippet"></div>

## What to do

* Use the `alt` attribute on the `<input>` element to provide a short text that describes the input element’s function.(e.g. `<input type="image" alt=“submit">`).
